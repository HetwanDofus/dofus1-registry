## Composition

`Console` is the root flex column and broadcasts the active `size` to descendants via context. The hierarchy is **three stacked rows** that match the FLA `UI_Debug` (Symbol 1153.xml):

- `ConsoleButtons` — dedicated 20 px header row on the rounded-top translucent black surface. Holds the four `WhiteCrossButton` instances at FLA-defined order: close (`X`), minimize (`_`), clear (`0`), copy (`C`).
- `ConsoleLogsArea` — the scrollable log row underneath. Collapses to `height: 0` when `size="minimized"`, expands to 200 px (`medium`) or 370 px (`max`). Same translucent black surface as the header — they read as one continuous rounded panel.
- `ConsoleLogLine` — one entry. The `level` prop picks the color channel — `"default"` (white `#FFFFFF`, normal info), `"error"` (red `#FF0000`), or `"info"` (green `#00FF00`, success). These map 1:1 to the three branches of `Kernel.showMessage` that write to `aks_a_logs` (Kernel.as:407-429); the original stores each line as an HTML `<font color>` tag and the widget just renders the concatenated HTML.
- `ConsoleInputRow` — rounded-bottom 20 px translucent white strip.
- `ConsolePrompt` — the prompt label. Children become the prefix; the component appends `" > "` (Debug.as:38-47).
- `ConsoleInput` — the command field. Submit on Enter, function-key callback on Tab / ↑ / ↓.

### Size states

| Prop value    | `ConsoleLogsArea` height | Total height |
| ------------- | ------------------------ | ------------ |
| `"minimized"` | `0px`                    | `40px` (20 header + 20 input) |
| `"medium"`    | `200px`                  | `240px` |
| `"max"`       | `370px`                  | `410px` |

`Debug.changeSize` (Debug.as:329-335) cycles `(current + 1) % 3` through `medium → minimized → max`. See `components/console-demo.tsx` for a working state machine that wires close / minimize / clear / copy / submit / history.

### Buttons

All four buttons render as 16 × 14 white pills with a single-character label, matching `WhiteCrossButton` + `ButtonFlatRoundDown` (DofusStylePackage.as:94 / core.swf id 1728). The FLA defaults:

| Component             | Label | AS button name |
| --------------------- | ----- | -------------- |
| `ConsoleCloseBtn`     | `"X"` | `_btnClose`    |
| `ConsoleMinimizeBtn`  | `"_"` | `_btnMinimize` |
| `ConsoleClearBtn`     | `"0"` | `_btnClear`    |
| `ConsoleCopyBtn`      | `"C"` | `_btnCopy`     |

Override the glyph via the `label` prop, or compose your own with the base `ConsoleButton` and any `onClick` handler.

### Wiring an instance

```tsx
function MyConsole() {
  const [size, setSize] = useState<ConsoleSize>("medium");
  const [logs, setLogs] = useState<string[]>([]);
  const cycle = () =>
    setSize((s) =>
      s === "medium" ? "minimized" : s === "minimized" ? "max" : "medium",
    );

  return (
    <Console size={size}>
      <ConsoleButtons>
        <ConsoleCloseBtn onClick={() => /* unmount */ {}} />
        <ConsoleMinimizeBtn onClick={cycle} />
        <ConsoleClearBtn onClick={() => setLogs([])} />
        <ConsoleCopyBtn
          onClick={() => navigator.clipboard.writeText(logs.join("\n"))}
        />
      </ConsoleButtons>
      <ConsoleLogsArea>
        {logs.map((l, i) => (
          <ConsoleLogLine key={i} recent={i === logs.length - 1}>
            {l}
          </ConsoleLogLine>
        ))}
      </ConsoleLogsArea>
      <ConsoleInputRow>
        <ConsolePrompt>debug</ConsolePrompt>
        <ConsoleInput
          onSubmit={(cmd) => setLogs((p) => [...p, `> ${cmd}`])}
          onFunctionKey={(e) => {
            if (e.code === "Tab") askShowAutoCompleteResult(api, e.currentPrompt);
            if (e.code === "ArrowUp") doConsoleHistoryUp(api);
            if (e.code === "ArrowDown") doConsoleHistoryDown(api);
          }}
        />
      </ConsoleInputRow>
    </Console>
  );
}
```
