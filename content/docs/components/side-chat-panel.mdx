## Installation

```bash
bunx shadcn@latest add side-chat-panel
```

## Composition

`SideChatPanel` is the root and flex-col container. Children are layout slots:

- `SideChatPanelFilters` — top header row (bg `#8C8368`, height `30px`). Holds `SideChatPanelFilter` checkboxes inside `SideChatPanelFilterGroup`, followed by `SideChatPanelActionButton` instances for **Search**, **Clear chat**, and the **move-panel** arrow — in that order. This matches the ship-order in the retroclient `D1Chat.html` (lines 249-268).
- `SideChatPanelMovePanelButton` — pulls current `side` from context and flips direction. Pass `onMove(next)` to persist. Placed last in the filters row. Only one arrow is visible at a time: the arrow points to the opposite side of `side`, so the user sees the direction the panel will move.
- `SideChatPanelInfoLog` — optional fixed-height info pane above the chat log.
- `SideChatPanelLog` — fills remaining height with a scrollable chat feed. Each child becomes a `retroChatMessage`.
- `SideChatPanelInputRow` — bottom input bar (bg `#D5CFAA`, height `40px`). Hosts `SideChatPanelPrefixButton` then `SideChatPanelTextInput` left-to-right.
- `SideChatPanelPrefixButton` — opens your chat prefix menu (wire to `ActionPopoutMenu`). Controlled via `prefix` prop.

### Channel palette

The filter and message colors come straight from `dofus/Constants.as:158-172`. Use the exports so you never hand-pick hex:

```tsx
import {
  SIDE_CHAT_CHANNEL,
  SIDE_CHAT_CHANNEL_COLORS,
  SIDE_CHAT_FILTER_CHANNELS,
} from "@/components/ui/side-chat-panel.channels";

// IDs mirror dofus.managers.ChatManager.TYPE_* (ChatManager.as:12-22).
SIDE_CHAT_CHANNEL.GUILD; // => 4

// Hex color as a CSS variable reference, themable via globals.css.
SIDE_CHAT_CHANNEL_COLORS[SIDE_CHAT_CHANNEL.GUILD];
// => "var(--color-side-chat-channel-guild)"   (resolves to #663399)

// The 8 channels that appear as filter checkboxes in the Retro layout.
SIDE_CHAT_FILTER_CHANNELS; // => [0, 2, 3, 4, 5, 6, 7, 10]
```

## Props

### `SideChatPanel`

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `side` | `"left" \| "right"` | `"right"` | Tracks which side the panel is rendered on. Used by `SideChatPanelMovePanelButton` to pick the arrow direction. |
| `theme` | `"light" \| "dark"` | `"light"` | Switches the scrollbar palette (light `#BEB998`/`#514A3C` vs. dark `#313131`/`#AAAAAA`). |

### `SideChatPanelFilter`

Extends `input` props (minus `type`, `color`).

| Prop | Type | Description |
|------|------|-------------|
| `color` | `string` | Channel color. Painted as the checkbox background (inline `backgroundColor` style). Unset falls back to `--color-side-chat-filter-bg` (black). |
| `label` | `string` | Tooltip shown via the `title` attribute — use the channel name. |

The ✔ mark is always rendered on top in `--color-side-chat-filter-check` (white) and toggles via `:checked`, so the colored box stays visible in both states.

### `SideChatPanelTextInput`

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `onSubmit` | `(value: string) => void` | — | Fired when the user presses any key in `submitKeys`. |
| `onFunctionKey` | `(e: SideChatPanelFunctionKeyEvent) => void` | — | Fires when the user presses any key in `functionKeys`. The event carries `key`, `code`, modifier flags, and `currentPrompt`. Hook history & autocomplete here. |
| `submitKeys` | `readonly string[]` | `["Enter", "NumpadEnter"]` | `KeyboardEvent.code` / `.key` values that trigger `onSubmit`. |
| `functionKeys` | `readonly string[]` | `["Tab", "PageUp", "PageDown", "ArrowUp", "ArrowDown"]` | `KeyboardEvent.code` / `.key` values that trigger `onFunctionKey`. Mirrors the Retro `onRetroChatFunctionKey` contract (keycodes 9, 33, 34, 38, 40). |

Both `submitKeys` and `functionKeys` match against `event.code` first, falling back to `event.key` — so you can pass either form. The defaults are exported as `SIDE_CHAT_PANEL_DEFAULT_SUBMIT_KEYS` and `SIDE_CHAT_PANEL_DEFAULT_FUNCTION_KEYS`.

### `SideChatPanelMovePanelButton`

| Prop | Type | Description |
|------|------|-------------|
| `onMove` | `(next: "left" \| "right") => void` | Receives the target side the user wants to flip to. |

## Notes

Mirrors the Retro launcher chat BrowserView (`D1Chat.html`). Colors come straight from the original:

- Panel bg: `#D5CFAA`
- Filters row bg: `#8C8368`
- Log background (outside content): `#8C8368` with `#D5CFAA` 10/5 px border frame
- Input row bg: `#D5CFAA`
- Input field bg: `#FFFFFF`

The widescreen panel flag itself is driven by AS2: `Kernel.as` sets `DisplayStyle = "widescreenpanels"` when `EnableWidescreenPanels` is true and Electron is enabled (`setWidescreenEnabled(true)`). The component does **not** own that decision — it's a pure layout primitive; the host shell (Electron / main window) decides when to mount it.
